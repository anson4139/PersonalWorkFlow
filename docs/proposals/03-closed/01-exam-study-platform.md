<details>
<summary>## 目標</summary>

建立一個可在手機上流暢閱讀與練習的考題學習平台，整合四科題庫，支援刷題、查看解析、追蹤學習進度，幫助備考「台灣金融科技力知識檢定」及「AI應用規劃師」三科。

</details>

<details>
<summary>## 範疇</summary>

**包含：**
- 四科題庫資料導入
  - 金融科技力（第10–21屆，~352 KB）
  - AI應用規劃師第一科（第114屆，~34 KB）
  - AI應用規劃師第二科（第114屆，~31 KB）
  - AI應用規劃師第三科（第114屆，~32 KB）
- 手機優先（Mobile-first）響應式介面
- 單題模式：逐題作答，點擊後揭曉答案與說明
- 瀏覽模式：按科目 / 屆次捲動閱讀
- 基本進度追蹤（已讀幾題 / 答對幾題）

**排除（初版）：**
- 使用者帳號 / 登入系統
- 後端 API / 資料庫（靜態或 localStorage 優先）
- AI 自動出題 / 錯題補強

</details>

<details>
<summary>## 設計</summary>

### 技術選型（已確認）

| 面向 | 決定 | 原因 |
|---|---|---|
| 前端框架 | **React + Vite (TypeScript)** | 多頁面互動、進度追蹤、日後擴充方便 |
| 樣式 | **Tailwind CSS** | Mobile-first 開發速度快 |
| 題庫儲存 | **JSON 靜態檔**（Python 腳本從 .md 轉換） | 前端讀取效能佳、易維護 |
| 進度儲存 | **localStorage** | 無需後端，資料存在瀏覽器本地 |
| 部署（第一階段） | **本機 `vite preview`** | 先在本地驗證功能與 UI |
| 部署（第二階段） | **`dashboard.buclaw.org`** + HTTP Basic Auth | 外部存取時保護題庫版權，避免搜尋引擎索引 |

### 資料流
```
原始 .md 題庫
  ↓ (轉換腳本)
JSON（subjects/<name>.json）
  ↓
前端讀取 → 渲染題目卡片
```

### 頁面結構（草稿）
```
首頁
  └─ 科目選擇（4 科 + 全部混合）
       └─ 屆次/章節選擇
            └─ 題目頁
                 ├─ 刷題模式（一題一題，點擊顯示答案）
                 └─ 瀏覽模式（全部展開）
```

### 手機 UX 重點
- 大字體、高對比選項按鈕
- 答案預設收折，點擊揭曉
- 底部導覽列（上一題 / 下一題 / 書籤）
- 避免橫向滾動

</details>
